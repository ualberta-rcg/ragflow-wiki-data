<!--T:1-->
Dummy Wheel exists in the wheelhouse to provide placeholders for some popular Python packages that are available on PyPI but as modules on our infrastructure. They are all tagged with a local version (<tt>dummy</tt>), for instance: <code>pyarrow-23.0.1+dummy.computecanada-py3-none-any.whl</code>.

<!--T:2-->
Such examples are:
* [[Arrow#PyArrow|pyarrow (Arrow)]]
* [[OpenCV#Python_bindings|opencv_python (OpenCV)]]
* [[MPI4py|MPI4py]]

<!--T:3-->
When a dummy wheel is selected by pip it will raise the error below since pip could not find a match for this requirement.

<!--T:4-->
= Troubleshooting =
== This is a normal error generated by this dummy wheel. ==

<!--T:5-->
{{Note|In the case below, <tt>pyarrow</tt> is used as an example but the same applies to other packages as well.}}

<!--T:6-->
When <tt>pyarrow</tt> dependency is not met, an error will be raised when the dummy wheel is selected. '''This is normal'''.
This means that either:
# the arrow module was not loaded, hence pip could not find pyarrow
# the arrow version loaded does not meet the requested version from the dependent package
# the arrow module loaded does not support the python version used

<!--T:7-->
=== Module not loaded ===
In this case, deactivate any virtual environment, load the module, re-activate the virtual environment, and re-run your pip install command.

<!--T:8-->
1. Deactivate your virtual environment : 
{{Command|test $VIRTUAL_ENV && deactivate}}

<!--T:9-->
2. Load the Arrow module :
{{Command|module load gcc arrow/x.y.z python/x.y.z}} 
where <tt>x.y.z</tt> is the version needed.

<!--T:10-->
3. Re-activate your previously created virtual env. :
{{Command|source <env>/bin/activate}} 

<!--T:11-->
4. And re-run your pip install command.
{{Command|pip install <package>}}

<!--T:12-->
=== Package/module version ===
The arrow module loaded does not meet the requirement. For instance, <tt>(from pyarrow>=21.0.0->datasets)</tt> which means that this version of the datasets package requires <tt>pyarrow</tt> greater or equal to version <tt>21.0.0</tt>.

<!--T:13-->
In other words, using any prior version with this version of datasets will results in the dummy wheel being installed.

<!--T:14-->
You can find the asked requirement (version) by looking at the line (<tt>pyarrow_noinstall</tt>)
{{Command|pip install --no-index datasets
|result=
...
Processing /cvmfs/soft.computecanada.ca/custom/python/wheelhouse/generic/pyarrow_noinstall-9999+dummy.computecanada.tar.gz (from pyarrow>=21.0.0->datasets)
}}

=== No Python bindings === <!--T:15--> 

<!--T:16-->
The loaded module may not have the python bindings for the python module loaded.
You can quickly verify by using:
{{Command|python -c "import pyarrow"}}
If the command displays nothing, the import was successful and the module provide <tt>pyarrow</tt> for the python module used.

<!--T:17-->
Or one can check if pip sees it:
{{Command
|pip list {{!}} grep pyarrow
|result=
pyarrow    23.0.0
}}

<!--T:18-->
If <tt>pip list</tt> show an entry, then <tt>pyarrow</tt> is available and seen by <tt>pip</tt>. Otherwise, in case of no entry, <tt>pyarrow</tt> is not available.
